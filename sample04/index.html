Staggerへのオブジェクト指定 2

staggerパラメータへオブジェクトで指定を行うことで、アニメーションを細かく指定することができます。
オブジェクト内には下記のプロパティを指定することができます。

grid

gridプロパティには配列指定と、定型値"auto"を指定できます。
要素の列がグリッド型に(整理された状態で)表示されている場合、"grid"プロパティへ幾つの行と列があるのかを(以下の様に)示すことで、 GSAPはその指定に応じて近接度合い(gridトゥイーン開始位置=起点)を計算することができます。
もしくは、grid:"auto" を使用することで、GSAPは要素の絶対座標位置を取得する element.getBoundingClientRect() 関数を使用して行と列を自動的に計算します。(レスポンシブレイアウトに最適)
もしも要素の列が整ったグリッドに配置されていない場合(ランダムであったり、不均一な配置の場合)には、 distributeByPosition()ヘルパー関数 が用意されているので、参考にしてみてください。

axis

axisプロパティには string値 "x"と"y" を指定することができます。
gridプロパティを定義したとき、 スタッガートゥイーンの伝播方向は from プロパティに指定されている(あるいはデフォルトの"start"の) x軸とy軸 の合計距離に基づきます。
axisプロパティを使用することで、x(横方向)、y(縦方向)のどちらか片方の軸に絞って スタッガートゥイーンの伝播方向を指定することができます。

ease

[String | Function]: The ease that distributes the start times of the animations. So "power2" would start out with bigger gaps and then get more tightly clustered toward the end. Default: "none".


アニメーション開始時のイージングを指定できます。 指定方法は通常のイージング指定と同じです。"Power4.out" こんな感じ。
デフォルトは"none"です。

repeat

Repeats immediately, not waiting for the other staggered animations to finish. If you put your repeat in the top level of the vars object of your tween like gsap.to(... {repeat:-1, stagger:{...}), it waits for all of the sub-tweens to finish before repeating the WHOLE sequence, but if you prefer to have each sub-tween repeat independently (so that as soon as each one completes, it immediately repeats itself), simply nest the repeat (and yoyo if necessary) inside the advanced stagger object, like gsap.to(... {stagger:{repeat:-1, ...}}); アニメーション開始時のイージングを指定できます。 指定方法は通常のイージング指定と同じです。"Power4.out" こんな感じ。
