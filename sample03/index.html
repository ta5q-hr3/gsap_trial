Stagger

"Stagger" 指定は連続した複数要素に指定した間隔(時間)で順番にアニメーション処理を実行します。
staggerプロパティには数値、オブジェクト({}での指定)、関数(function)を指定することができます。
例えば、 stagger: 0.1 を指定すると0.1 秒間隔で「渡したリスト(要素が取得された)順」にアニメーションを実行します。

Staggerへのオブジェクト指定

staggerパラメータへオブジェクトで指定を行うことで、アニメーションを細かく指定することができます。
オブジェクト内には下記のプロパティを指定することができます。

amount

staggerする各アニメーション間隔の合計量指定プロパティです。秒単位で指定します。
指定値(秒数)を対象の要素数で分割した待機時間が各アニメーションの開始タイミング前に追加されます。 つまり、待機時間を総量で指定します。
例えば、100個並んだ要素へのstaggerに amount:1 を指定した場合、1秒を1/100にした = 0.01秒の待機がアニメーション開始前に追加されます。 以下の例では stagger: {amount:2} を指定しているため、各アニメーションのスタートに 2/8(=0.25)秒の遅延が発生しています。

amount : 2

amount : 2

each

staggerする各アニメーション間隔の直接指定プロパティです。秒単位で指定します。
指定値(秒数)の待機時間を対象要素のアニメーション開始タイミング前に追加します。つまり、個々のアニメーションの待機時間を指定します。
例えば、100個並んだ要素へのstaggerに each:1 を指定した場合、100回のアニメーションの前に1秒の待機がアニメーション開始前に追加されます。 (つまり、指定した待機時間x対象要素数分の追加時間が生まれます)
以下の例では stagger: {each:1} を指定しているため、各アニメーションのスタートに1秒の遅延が発生しています。

from

[String | Integer | Array]: The animation for each element will begin based on the element's proximity to the "from" value in the Array (the closer it is, the sooner it'll begin). You can also use the following string values: "start", "center", "edges", "random", or "end" ("random" was added in version 3.1.0). If you have a grid defined, you can specify decimal values indicating the progress on each axis, like [0.5,0.5] would be the center, [1,0] would be the top right corner, etc. Default: 0. 各要素のアニメーションは、その要素が「from」に近いかどうかに基づいて開始されます。 各要素のアニメーションは、配列の "from" 値への要素の近さに基づいて開始されます。(に近いほど、早く開始されます。） 以下の文字列値も使用可能です。"start"、"center", "edges"、"random"、"end" (バージョン 3.1.0 で "random" が追加されました)。 グリッドが定義されている場合、各軸の進行状況を示す10進数値を指定することができます。0.5,0.5]が中心、 [1,0]が中心というように、各軸の進捗を示す10進数値を指定できます。[1,0]は右上隅、といった具合です。 デフォルトは0です。

Staggerを指定する配列の中で、アニメーションを開始する位置のプロパティです。

grid

[Array | "auto"]: If the elements are being displayed in a grid visually, indicate how many rows and columns there are (like grid:[9,15]) so that GSAP can calculate proximities accordingly. Or use grid:"auto" to have GSAP automatically calculate the rows and columns using element.getBoundingClientRect() (great for responsive layouts). Grids are assumed to flow from top left to bottom right layout-wise (like text that wraps at the right edge). Or if your elements aren't arranged in a uniform grid, check out the distributeByPosition() helper function we created.

axis

[string]: If you define a grid, staggers are based on each element's total distance to the "from" value on both the x and y axis, but you can focus on just one axis if you prefer ("x" or "y"). Use the demo above to see the effect (it makes more sense when you see it visually).

ease

アニメーション開始時のイージングを指定できます。 指定方法は通常のイージング指定と同じです。"Power4.out" こんな感じ。
