Stagger 1

"Stagger" 指定は連続した複数要素に指定した間隔(時間)で順番にアニメーション(トゥイーン)処理を実行します。
staggerプロパティには数値、オブジェクト({}での指定)、関数(function)を指定することができます。
例えば、 stagger: 0.1 を指定すると0.1 秒間隔で「渡したリスト(要素が取得された)順」にアニメーションを実行します。

Staggerへのオブジェクト指定

staggerパラメータへオブジェクトで指定を行うことで、アニメーションを細かく指定することができます。
オブジェクト内には下記のプロパティを指定することができます。 

      gsap.from(
        variable,
        {
          ...
          stagger: {
            amount:2,
            each:0.1,
            from:[0.3,0.4],
            grid:"auto",
          }
        }
      );

amount

staggerする各アニメーション間隔の合計量指定プロパティです。秒単位で指定します。
指定値(秒数)を対象の要素数で分割した待機時間が各アニメーションの開始タイミング前に追加されます。 つまり、待機時間を総量で指定します。
例えば、100個並んだ要素へのstaggerに amount:1 を指定した場合、1秒を1/100にした = 0.01秒の待機がアニメーション開始前に追加されます。 以下の例では stagger: {amount:2} を指定しているため、各アニメーションのスタートに 2/8(=0.25)秒の遅延が発生しています。

each

staggerする各アニメーション間隔の直接指定プロパティです。秒単位で指定します。
指定値(秒数)の待機時間を対象要素のアニメーション開始タイミング前に追加します。つまり、個々のアニメーションの待機時間を指定します。
例えば、100個並んだ要素へのstaggerに each:1 を指定した場合、100回のアニメーションの前に1秒の待機がアニメーション開始前に追加されます。 (つまり、指定した待機時間x対象要素数分の追加時間が生まれます)
以下の例では stagger: {each:1} を指定しているため、各アニメーションのスタートに1秒の遅延が発生しています。

from

Staggerを指定する配列の中で、アニメーションを開始する位置のプロパティです。
Integer(数値)、String(定型値)、Array(配列)で指定します。

Integer(数値)での指定

特定の要素からアニメーションを開始するには、対象配列内の要素indexを指定します。 例えばfrom:4 を指定すると、対象配列の５番目の要素からアニメーションを開始します。
※ javascriptのArrayIndexに則って[0]から始まるためです。
要素indexで指定した要素に近い順にアニメーションを実行します。上記例のIndex:4の次にはIndexが3と5の要素のアニメーションが実行されます。

String(定型値)での指定

定型のString値で開始位置を指定することができます。
"start"、"center", "edges"、"random"、"end" の５つの定型値を指定することができます。

from:"start" を指定すると、対象とした配列の先頭(index [0])からアニメーションさせます。デフォルトの指定です。

例えば、要素8つの配列の場合、中央のIndexが4,5の要素から同時にアニメーションを開始します。

Array(配列)での指定

後述する grid: 指定を併用している場合、トゥイーンの起点を fromプロパティへx軸、y軸の深度を示す値(0〜1)の配列で指定することができます。 

          from : [x,y]

たとえば from : [0.5,0.5] を指定すると中心から、[1,0]は右上、[0,1]は左下です。 デフォルトの値は0 ([0,0]...左上) です。
要するに、gridプロパティにおけるアニメーション開始位置の指定にfromプロパティを使用することができます。
